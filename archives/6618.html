黄金交易风险控制

03 December 2010 Categoried in Finance 0 Preview : 327 , Read : 527 Index : 3747

== 业务风险 ==

=== 政策风险、交易风险和库存持仓风险 ===

首先，我行本身是上海黄金交易所金融类会员，根据规则可以开展黄金自营业务、代理机构业务和代理个人黄金业务。人民银行政策允许银行开展代理黄金业务。所以不存在政策风险。

其次，个人黄金延期业务定位于代客黄金买卖模式，仅收取手续费并不承担资金和黄金价位波动的风险，在交易过程中银行不承担任何责任，委托价格和数量有客户自主决定，成交与否由上海黄金交易所撮合决定。无论是否成交，银行开展代理业务都不承担风险，所以不存在交易风险。

再次，上海黄金交易所对库存和持仓管理到每个客户，每天日终清算时将提供客户的库存和持仓明细以进行核对。同时在交易过程中如果客户库存或者持仓不足，上海黄金交易所会直接进行无效处理并告知客户交易无效。所以不存在库存持仓风险。

因此，我行开展个人黄金延期业务风险面临的最主要风险是客户准入风险和资金不足的风险。

=== 客户准入风险 ===

客户准入风险是：客户申请黄金延期交易资格时，对延期交易具体内容以及杠杆交易风险的确认过程。

客户准入风险的规避方法是我行对申请客户进行风险评级和充分的风险揭示，低风险承受能力的客户将被我行主动拒绝。

=== 资金不足风险 ===

资金不足风险是：由于上海黄金交易所对客户资金实行二级清算制，客户交易的资金和费用由会员进行清算和代收代缴，上海黄金交易所只向会员清算资金，因此客户的资金风险将由银行承担。

具体来说，客户资金风险主要有以下四类：

==== 交易资金风险 ====

交易资金风险是指当客户进行黄金买卖、开仓时，交易所不验证客户资金是否够足够，只要我行的资金足够，而直接扣除我行资金。

因此，客户在可用资金不足的情况下买卖、开仓，会给我行带来资金风险。防范交易资金风险的方法是当客户在买卖、开仓时，需要及时验证客户的可用资金是否足够。

当可用资金 > [委托价×手数×每手单位重量×（手续费率+保证金率）]时，客户才可以进行买入、开仓。如果可用资金不足，客户不能进行买入、开仓操作。

同时为了保证客户的资金充足度，必须对客户的资金进行冻结处理，以防止客户资金挪作他用。

==== 各种费用风险 ====

各种费用风险是指客户进行黄金交易的费用包括延期补偿费、升贴水、仓储费、运保费、溢短费等，这些费用在日终清算时根据结算价格或者交割结果产生。如果客户在日终清算时资金不足就会导致银行垫资，造成资金风险。

各种费用规避的方法是对费用分成两类进行资金预冻结：

1、提供保证金预冻结：用于对运保费和溢短费用进行预冻结处理。

2、仓储费保证金预冻结：用于对客户库存的仓储费进行预冻结处理。

通过冻结可以防止客户在资金不足时由于清算发生的费用而造成的银行资金风险。

==== 交割违约风险 ====

交割违约风险是指黄金保证金交易合约的交割采用实物交收申报制度，交收申报成功，在当日清算时完成资金和实物的过户。在交收申报成功后当日清算时，交货的客户要扣除库存，收货的客户扣除资金（扣除资金=交割成交重量×当日结算价）。由于交割清算都是由交易所和会员清算。在交割过程中，如果收货客户的资金不足，交易所不会关心客户资金是否足够，而会直接扣除我行的保证金，造成我行垫资的风险。

交割风险规避的方法是在收货客户交收申报时就冻结客户交割保证金。交割保证金 =当日最新加权均价×申报手数×每手数量×(交割保证金率+交收价格浮动比率-买保证金率)。交割保证金冻结后，在客户交割成交时从交割保证金中扣除所需资金，释放多余资金，以防止我行的交割风险。

==== 持仓盈亏风险 ====

持仓盈亏风险盈亏由于客户持仓在价格波动的情况下造成，持仓盈亏风险主要包括：逐日盯市盈亏、平仓盈亏，浮动盈亏。其中逐日盯市盈亏需要结算资金，而浮动盈亏仅仅反映客户交易过程中的风险情况。平仓盈亏是由于价格波动客户平仓时造成的，也会占用客户资金。由于交易所仅仅对银行结算资金而不对客户结算资金，因此当客户资金不足时由我行承担资金风险。
在开展黄金保证金交易时，持仓盈亏风险是最重要的风险。因为盈亏根据价格的波动产生和客户持仓品种、方向产生，具有很大的随机性和不可预测性。影响客户持仓盈亏的主要因素是客户的持仓情况（品种和方向）、可用资金的充足度以及价格波动造成的盈亏情况。通常可以采取下面的手段预防客户的持仓盈亏风险：

1、价格波动的涨跌停板限制；

2、保证金高于上海黄金交易所的要求，并且根据风险情况及时提高保证金率；

3、对客户的浮动亏损实时计算并且在客户的可用资金和可提取资金中预冻结；

4、对客户实行限仓制度；

5、通过风险监控系统，对客户的综合风险度进行实时监控和预警；

6、当客户发生风险时进行强行平仓处理。

== 持仓风险控制 ==

持仓盈亏的风险可以由黄金延期交易风控操作员结合黄金延期业务的风险管理软件系统来完成。风险管理软件系统的最主要功能是综合考虑客户持仓保证金、可用资金和客户浮动亏损之间的联系，对客户的风险情况进行实时监控和实时评估。同时根据客户的资金情况，风险程度等采用成熟的公式和模型对客户风险进行量化后计算并给出相应的风险控制处理措施。

风控操作员拥有风险控制的最终决定权。风险管理系统可以辅助风控操作员完成风险控制，但不能替代风控操作员的最终决定权。

=== 风险控制点 ===

1)事前风险试算

风险试算为银行提供了在未来设定的价格变化范围内，对需采用的保证金率的预测能力。通过风险试算可以预知当价格波动时，怎样的保证金率可以协助客户有效抵御风险。
客户的资金亏损是由于持仓在一定行情波动下造成的，而持仓保证金则是抵御风险的第一道防线。保证金比率的合理设置将对银行抵御黄金延期业务风险起到至关重要的作用，尤其是伴随法定长假、常常出现价格的大起大落，更加需要通过临时调高保证金率的手段来抵御风险。

2)风险实时预警

风险计算的目的是为了实现风险预警。客户风险受到资金、行情、持仓等多种因素的影响，通常情况下客户的风险来自于这些因素的综合影响。通过风险计算对客户的风险程度进行量化以后，就可以及时预警和规避客户的风险。

3)发现风险后的处理

当客户产生风险时有两种处理方法，一种是要求客户追加保证金，另外一种是在追加不成功的情况下进行强行平仓处理。
在实际情况下，客户的风险是由于一系列持仓及综合因素造成的，根据客户的风险程度需要计算一个合理的平仓组合，对客户的持仓序列进行处理。风控系统将提供对强平单生成和自动化处理功能的支持。

=== 二级系统主要风险控制功能 ===

1)限仓控制

分合约设置客户持仓限额，防止持仓过高产生风险，开仓交易时检查，当持仓达此限额时禁止开仓。
开仓交易时检查此限额。

2)保证金率调整

系统支持在对保证金率进行调整，修改后的交易按照新的费率收取保证金，修改前的交易收取的保证金不变，清算时按照调整后的费率计算客户持仓保证金。

3)交易权限控制

操作员可以手工控制客户交易权限，正常、冻结及挂失。委托交易时要对客户交易权限进行检查。

=== 风险参数及风控公式 ===

风险分段设置

要求每套指标包含三种风险等级定义：1-绿色安全等级，2-黄色追保等级，3-红色强平等级。
达黄色追保等级的客户要对其计算“追保金额”，追加此金额可使客户在目前行情下风险指标降至绿色安全等级或以下。
风控操作员生成强平单时，操作的客户为达红色强平等级的客户。计算强平数量（金额）的依据为以强平指令最差价格成交后，此客户的风险指标降至绿色安全等级或以下。

风险度计算

选择合适的时机计算风险度，在保证金跟风险度变化的同时，可以在客户量很大的情况下，也保持足够的效率，防止系统过载。系统设计中，包含了两个风险度R1和R2，两者互为补充。

当客户的可用资金、持仓保证金、浮动盈亏发生变化时，会对客户的风险度产生影响。又因为当现货、递延、交割收货、中立仓收货成交时或出入金时可用资金会发生变化；当递延成交时持仓保证金会发生变化；当递延成交时或递延合约价格波动时浮动盈亏会发生变化；所以任何合约成交回报时、递延合约价格波动时、出入金时都要触发风险度的计算。

系统需有4个风险度计算触发点：任何合约成交时、递延合约价格变化时、出入金时、定时。任何合约成交或出入金时对发生交易的客户计算风险度；递延合约价格变化时对持有相应合约持仓的客户计算风险度；定时作为补充，对全部持仓客户计算风险度，主要用于防止意外情况导致的计算不及时。计算均为后台自动运行，无需界面。

风险度1 = （持仓保证金）/（持仓保证金+可用资金+浮动盈亏）

注意：

计算时，当中间结果分母为< =0时，或风险度计算值>100时，将风险度置为最大值100。(此处将最大值设为100是因为根据定义，风险度为100意味着：持仓保证金数额已经是全部平仓后资金的100倍，再大已没有分别)

持仓保证金 = ∑(历史开仓合约数量×上日合约结算价×合约保证金率)＋∑(今日开仓合约数量×合约成交价×合约保证金率)

合约保证金率 = 金交所合约保证金率＋会员合约保证金率

浮动盈亏 =∑(历史买开仓合约数量×(合约当前价－上日合约结算价))＋∑(历史卖开仓合约数量×(上日合约结算价－合约当前价))＋∑(当日买开仓合约数量×(合约当前价－合约开仓价))＋∑(当日卖开仓合约数量×(合约开仓价－合约当前价))

当风险度1>1时：

追保金额 = 持仓保证金 / 绿线风险度 － (持仓保证金+可用资金+浮动盈亏)

当绿线风险度 =1时

追保金额 = －(可用资金+浮动盈亏)

=== 风控公式比较分析 ===

C—可用资金，B—浮动盈亏，M1—会员保证金（即会员多收的持仓保证金），M2—交易所保证金，R1—风险度1, R2—风险度2

公式一：R = (M1+M2) / (C+B+M1+M2)

公式二：R = － (C+B) / M1

在实际业务中，对于会员比较关注的风险点有两个：

客户的资金情况不满足会员的业务要求，即客户的“浮动亏损”将“可用资金”全部抵消，此时B = －C。
此风险点的临界值，对于公式一，R=1；对于公式二，R=0。

客户的资金情况不满足交易所的业务要求，即客户的“浮动亏损”将“可用资金”和“会员保证金”全部抵消，此时B = －(C+M1)。此风险点是最危险的，是会员的代理风险之所在。
此风险点的临界值，对于公式一，R = (M1+M2) / M2 = 1 + M1 / M2；对于公式二，R=1。

可用资金并不含浮动盈亏因素，只有每日清算时才加进来，盘中无论盈亏都不反映在可用资金里面，系统有“盈亏后可用”来反映

可以看到，当客户的资金情况不满足交易所的业务要求时，公式一中的R值大于（1 + M1 / M2），然而（1 + M1 / M2）的值取决于M1 / M2的比值。下面分四种情况具体说明：

情况一：

如果客户只做一个合约，例如Au(T+D)，且会员的多空保证金率一致。此时1 + M1 / M2也是固定值，那么公式一的强平线设置一个固定值是可以实现的。

情况二：

如果客户只做一个合约，例如Au(T+D)，且会员的多空保证金率不一致。1 + M1 / M2的值，不仅受“会员多头保证金率”的影响，还受“该合约的多空持仓量”的影响，然而“合约的多空持仓量”是变化的，那么公式一的强平线设置一个固定值是不可以实现的。

情况三：

日终时，延期合约的“多头持仓均价”等于“空头持仓均价”，即为合约的“结算价”。1 + M1 / M2的值，不仅受“会员保证金率”的影响，还受“各个合约的多空持仓量”和“各个合约的结算价”影响，而每天日终后“各个合约的多空持仓量”和“各个合约的结算价”是变化的，那么公式一的强平线设置一个固定值是不现实的。

情况四：

盘中交易时，延期合约的“多头持仓均价”和“空头持仓均价”是分别变化的，并不相等。1 + M1 / M2的值，不仅受“会员保证金率”的影响，还受“各个合约的多空持仓量”和“各个合约的多空持仓均价”影响，而盘中的“各个合约的多空持仓量”和“各个合约的多空持仓均价”是变化的，那么公式一的强平线设置一个固定值更是不现实的。

公式二的实现简述：

公式二中舍弃了M2，实际上对于会员衡量风险来讲，没必要关心M2的取值。一旦客户的“浮动亏损”将“可用资金”和“会员保证金”全部抵消，会员就要承担风险，这也是会员代理客户做延期交易的风险之所在。

公式二衡量了客户将“会员保证金”的亏损程度，是会员衡量风险的核心，也应该是二级风控系统的核心。前面已经分析过，公式二中清晰的描述出了会员关心的两个风险点：

客户的资金情况不满足会员的业务要求。此时R＝0；

客户的资金情况不满足交易所的业务要求。此时R＝1。

而公式一在描述第二个风险点时，会遇到障碍的，它是一个浮动值，受前面所分析的各种因素的影响，在业务操作上是没有可行性的。

二级系统如使用公式二来衡量客户的风险，在前两个风险点的基础上会增加“追保计算线”和“强平控制线”的概念，此两线是允许会员进行配置的，假设追保计算线为W，强平控制线为U，则W≤0，0≤U≤1。

系统风险控制简述

当客户的R大于0，进入会员保证金不足区域，系统产生追保通知。

当客户的R大于“强平控制线”，系统产生强平警告，由风险监控员来决定是否对客户进行强平。

当客户的R大于1，系统产生风险严重警告，由风险监控员来决定是否对客户进行强平，通常此时建议风险监控员必须强平，否则会员则承担风险。

计算客户的“追保金额”时使用“追保计算线”，系统提供允许多向客户进行追保的功能，会员视情况使用，通常建议“追保计算线”配置为0。

使用“自动生成强平单”功能，系统按既定规则生成强平单，以满足强平后使客户的R值降为0或0以下。

公式二不足之处：不能设置“会员保证金率”为0，必须为大于0的值，实际上这是符合银行会员的要求的，不存在矛盾之处。
