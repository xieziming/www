QTP Action 参数传递

17 September 2012 Categoried in 测试 0 Preview : 4320 , Read : 1920 Index : 9026

以QTP的Flight为例子讲下Top Level的Action参数传递。

SystemUtil.Run "C:\Program Files\Mercury\QuickTest Professional\samples\flight\app\flight4a.exe","","C:\Program Files\Mercury\QuickTest Professional\samples\flight\app\","open"
Dialog("Login").WinEdit("Agent Name:").Set Parameter("InAction1")
wait 5
Dialog("Login").WinEdit("Agent Name:").Type  micTab
Dialog("Login").WinEdit("Password:").SetSecure "4649c633ffc8803c10097292953c6334fde3e923"
Dialog("Login").WinEdit("Password:").Type  micReturn
Window("Flight Reservation").Close
Parameter("OutAction1") = True

然后，鼠标选中Keyword View中的Action1，点右键—Action Property，在Parameters的Tab标签下，分别加入：输入参数 InAction1 ，类型String；输出参数 OutAction1，类型 Boolean。

再然后，在QTP的菜单File—>>Settings的Parameters的Tab标签下，分别加入：输入参数 InParam1 ，类型String；输出参数 OutParam1，类型 Boolean。

接着，鼠标还是选中Keyword View中的Action1，点右键，这次点“Action Call Properties”，在Parameter Values里进行参数化传递设置，把InParam1的值传递给InAction1，把OutAction1的值传递给OutParam1。

以上设置完毕后，点“保存”，保存到C:\下，存为Test1好了。

最后，在你的硬盘上新建一个vbs文件，文件内容如下：

Dim qtApp ,pDefColl,pDef ,rtParams,rtParam 
Set qtApp = CreateObject("QuickTest.Application") 
qtApp.Launch 
qtApp.Visible = True 
qtApp.Open "C:\Test1" 
Set pDefColl = qtApp.Test.ParameterDefinitions 
'cnt = pDefColl.Count 
'Indx = 1 
'While Indx < = cnt 
    'Set pDef = pDefColl.Item(Indx) 
    'Indx = Indx + 1 
'Wend 
Set rtParams = pDefColl.GetParameters() 
Set rtParam = rtParams.Item("InParam1") 
rtParam.Value = "songfun" 
qtApp.Test.Run , True, rtParams
MsgBox rtParams.Item("OutParam1").Value

做完这步之后，保存这个vbs文件，双击执行这个vbs文件，你会发现它自动启动了QTP，而且进行了自动测试，最后还取到了运行成功与否的布尔值。

它的参数的整个传递过程是：

外部vbs文件 传参数给QuickTest的Test的输入参数InParam1，然后InParam1传参数到InAction1去驱动了Action1的测试，然后通过这个Action1得出了OutAction1的值，然后通过OutAction1传给OutParam1，最后再传回到vbs文件中。示例用MsgBox来打出重新传回到vbs文件中的字符串。

除了Action传给程序顶部外，我们还关注Action之间的参数传递：

1. 同级别Action间参数传递.
2. 嵌套式Action间参数传递.

下面举个例子讲下同级别Action间参数传递：

我们先将一个大的action分为多个action

Action1：用户登录 + 点击添加功能 + 数据库检查（没有添加信息前的信息） +
Action2：添加页面添加数据 + 提交 + 系统判断 +
Action3：数据库判断（点击提交后的数据信息） +退出系统

RunAction "Action1", oneIteration
RunAction "Action2", oneIteration
RunAction "Action3", oneIteration

如果我们在Action2中想用到Action1中的某个变量值，这时要怎么办呢？

（1）我们先在Action1中添加如下代码

Parameter(“action2_needed”)=randomnumber(1,4)
意思是将randomnumber(1,4)的值赋给输出参数action2_needed;

然后在Keyword view中选中Action1添加1个输出参数action2_needed。

（2）在Keyword view中选中Action2添加1个输入参数action2_needed。然后先在Action2中添加如下代码

Dim B
B=parameter(“action2_needed”)
Msgbox B

(3)主action应该变成如下

dim sTemp
RunAction "Action1", oneIteration,sTemp'把Action1里的Output Parameter(action2_needed)传递给了sTemp 
RunAction "Action2", oneIteration,sTemp'把sTemp传递给了Action2的Input Parameter(action2_needed)
'RunAction中传递的参数名称和我们在界面中填入的传入/传出参数的名称无关，可以任意命名，只与其在QTP界面中传入\传出参数的顺序有关(传入参数在前传出参数在后)。也就是说RunAction中传递的第一个参数对应QTP界面中传入参数的第一个(当然如果没有传入参数的话指的就是传出参数的第一个)。然后按照先传入参数后传出参数的顺序依次对应RunAction相应的值。

下面举个例子讲下嵌套式Action间参数传递：

假设Action1是Action2的父Action。Action2接收Action1中传出参数。
（1）Action1中的代码：

Dim a 
a="hello,moon_wang!"
runaction "Action2", oneIteration,a ‘调用Action2，并将a的值传给action2中的传入参数action2_needed

（2）我们在Action1中无需任何设置，因为它不需要传给外部的Action用，Action2在其内部。在Action2中的设置一个输入参数action2_needed。

（3）Action2的代码如下：

Dim b
b=parameter("action2_needed") ‘B从传入参数c中取道由action1传过来的值“hello,moon_wang!”
msgbox b

下面再举个例子讲下将对象的属性变成Action参数传出去：

比如我们要将Action1出现的一个文本框中的值（比如订单号），传递给Action2中作为查询条件。

（1）在Action1中添加一个输出参数action2_needed。

（2）在Keyword view中选中要传出属性的对象（比如文本框），右键，在菜单中选择”Insert OutPut Value…”，打开OutPut value Properties 窗口。选中属性text前的复选框(打钩)，然后点击 Modify 按钮，在Output types:中选择 Test/action parameter,在Parameter选择action2_needed(这就是刚才Output中添加的变量),然后,点击”OK”,”OK”保存.

（3）在Action2中添加一个输入参数action2_needed。

Window("Flight Reservation").Dialog("Open Order").WinEdit("Edit").Set Parameter("action2_needed")
