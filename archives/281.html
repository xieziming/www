答辩压力

13 December 2008 Categoried in 点滴 0 Preview : 81 , Read : 787 Index : 1672

随着答辩日期的临近，倍感压力啊，虽然6月过去了，但还是有很多东西不懂。最近国内的答辩，看他们被问得像*头样出来，还真有点心惊肉跳，那些专家组的问题得真BT！虽然过是肯定能过的，但最好不要太早被问倒，FACE要紧啊。说实话，这个答辩我还真的没非常认真准备，因为其他要看的东西实在太多，加上近来身体不适，就把我在实验室做的一个例子拿出来讲，《实验室两纤双向复用段环扩环专题》。

下面以在实验室环境下由S385-1、S390-1和S390-2组成的二纤双向复用段环为例来说明扩环一般步骤：

图1：是扩环的示意图，在节点S390-1、S390-2之间插入S330-1

另外，为了检验扩环过程中是否断业务，我在在S390-1与S390-2分别加了块SFE板，两点之间建个虚拟局域网，各接一台PC，互ping，ping的方式为ping *.*.*.* -t，即一直ping，看操作过程是否断业务。

扩容前：
1、查询待扩容复用段环速率等级,各站点的告警、性能以及业务配置情况，要求没有异常告警和性能事件；且环上没有任何倒换存在；
2、备份网管数据库；
3、在扩容前反复测试复用段倒换，确认复用段环倒换正常；
4、对待扩容网元S330-1 NCP初始化完毕，网元网管数据配置无误；
5、准备扩容需用到的尾纤和相关备板，网管扩容后网络拓扑草图(规划网元连接的光板槽位对应关系)。
6、保证PC1与PC2相互能ping通。

扩容过程：

第一步 倒换被扩节点之间的业务：

在S390-2去S390-1的光板上插入MS-AIS告警，使S390-2和S390-1发生复用段保护倒换，即把S390-2与S390-1之间的业务通过环保护倒换到保护通道中，此时，PC1与PC2相互能ping通，说明业务没断。

保护倒换完成后，对S390-1和S390-2下发APS暂停命令。然后断开S390-2、S390-1之间的本复用段使用的光纤连接，连接好S390-2与S330-1之间的光纤，S390-1与S330-1之间的光纤。

第二步 离线删除每个节点的复用段保护关系：
修改该复用段保护组的每个网元为离线，即S390-1、S390-2、S38-1为离线。记录复用段保护组的ID、各个网元的APS ID、使用的APS字节、复用段保护中的工作单元端口保护单元端口及它们之间的连接关系、已经配置的复用段保护属性。（为后面建立几乎一样的复用段保护组用）
选中S390-2、S390-1、S385-1，删除每个网元的复用段保护关系, 再删除掉该复用段保护组。

第三步 离线建立基本与原复用段保护组相同的复用段保护关系：

1.离线删除S390-1与S390-2之间的光纤连接关系。
2.离线建立S390-2与S330-1，S330-1、S390-1之间的光纤连接关系。
3.创建一个与扩环前相同保护组ID、除新增S330-1外其他节点APS ID不变、除新增S330-1外其他节点的复用段工作单元保护单元连接关系不变的复用段保护组（包含S330-1），修改复用段保护属性与扩环前相同。对S330-1配置复用段保护的工作单元与保护单元的连接关系。点击“确认”，再点“应用”。

第四步 配置节点4的时隙：

由于S390-2与S390-1之间的时隙需要在S330-1穿通，因此在S330-1配置好穿通时隙。

第五步 在线下载数据库：

选中所有节点网元，即S390-2、S390-1、S385、S330-1，将它们由“离线”修改为“在线”，使网管能正常管理上所有网元。此时PC1与PC2相互能ping通，业务没断。对S390-2、S390-1、S330-1下载数据库，包括“网元连接”(需添加环境变量ZTELINK，取值为：1)、“复用段保护设置”、“复用段保护组信息”、“复用段保护属性信息”、“复用段保护倒换”、“复用段TOPO”、“APS参数”、“APS功能设置”，这8个数据库。

另外，对S330-1需要下载“时隙子网保护设置”。由于S385-1的数据没有变化，所以可以不下载数据。

第六步 启动APS：

取消掉插入的MS-AIS告警。启动各个节点APS协议。等待S390-1、S390-2自动撤销复用段的保护倒换，业务经过S330-1走工作通道。整个过程PC1与PC2相互能ping通，业务不中断。

总结：在线插MS-AIS倒换→在线暂停APS→……→离线修改→在线下载→在线撤销MS-AIS→在线启动APS。

English Topic：

Working Mode of Slave Node Clock in Master-Slave Synchronization Network

The clock synchronization data must be configured for all kinds of communication networks. In the network, each node should track the same clock source to obtain clock synchronization. Clock synchronization plays a significant role in telecom network, and timing is one of the most fundamental operations in communication. There are three types of working modes for the clocks in the Slave node clock (lower level node) in master-slave synchronization network.

1. Lock Mode
The clock reference that the Slave node clock tracks and locks comes from the upper level node. This clock can be one of the following three types: the primary reference clock (PRC) in the network, the clock sent by the embedded clock source in the upper level node, and the GPS clock in local area. Compared with the Hold mode and the Free Run mode, the Lock mode for Slave node clock has the highest degree of precision.

2. Hold Mode
The Slave node clock enters the Hold mode when all the timing references are lost. At this stage, the clock source in the Slave node clock uses the latest frequency information stored in the timing reference signal before it is lost. It means the clock has the Memory function, by which it can provide timing signals that is closest to the former timing reference. The working methodology of the Hold mode is to keep the frequency deviation between the frequency of the Slave node clock and that of the primary reference clock as small as possible in a period as long as possible; however, the intrinsic oscillation frequency in the oscillator wanders gradually, and the clock adopting the Hold mode cannot provide a high degree of precision in the long term. Because of it, the precision of the Slave node clock adopting Hold mode is lower than that adopting the Lock mode.

3. Free Run Mode
When the clock lost all the external timing references, and it also lost the memory for timing reference or it has been in the Hold mode for too long a period, the oscillator enters the Free Run mode.
The Free Run mode has the lowest degree of precision among the three modes, and this mode is adopted when all else failed.
In the communication network, data and image services have higher requirement on clock precision than voice service does. However, when the clock precision deteriorates to a certain degree, the voice service is adversely affected ( noise occurs in the communication ).
